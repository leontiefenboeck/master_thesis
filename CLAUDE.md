# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository layout

- `code/` — Python research code (PyTorch). The experiments are run from `code/` directly; there is no package layout, `setup.py`, or test suite. Imports are flat (e.g. `from gmm import GMM`).
- `thesis/` — LaTeX source for the master's thesis (Karl Voit template). Build with `cd thesis && make` (uses `pdflatex` + `biber`). Output is `thesis/main.pdf`.
- `idea.pdf` — original project proposal / problem statement.

## Running experiments

There is no test runner. Experiments are scripts you execute directly:

```
cd code && python demo.py
```

`demo.py` is the end-to-end entry point: it generates synthetic data via `DataManager`, fits a `GMM` density estimator and a sklearn `LogisticRegression`, constructs a partially observed test point, and compares the two expectation estimators (`MCBaseline` vs `PCPG`) using `utils.evaluate`.

`circuit.py` is a separate, self-contained experiment using the `cirkit` library — it does not share the `GMM`/`PCPG` pipeline. Treat it as exploratory scratch code.

Figures generated by `utils.plot_2D` etc. are written to `code/figures/`.

## Core architecture

The thesis investigates computing **E[σ(wᵀx) | x_obs]** — the expected logistic-regression prediction marginalised over missing features — under a learned density p(x). Two estimators are compared:

1. **`MCBaseline`** (`baselines.py`) — vanilla Monte Carlo: draw x_mis from the conditional p(x_mis | x_obs) and average σ(wᵀx).
2. **`PCPG`** (`pcpg.py`) — Pólya-Gamma / characteristic-function estimator. Uses the Gaussian integral identity for σ via Pólya-Gamma augmentation: draws ω ~ PG(1,0) and u ~ N(0, ω), then evaluates the conditional characteristic function φ(s) of the GMM at s = −u·w_mis. This is the contribution of the thesis; the math in `PCPG.pg_expectation` mirrors the derivation in `idea.pdf`.

Both estimators expect the density model to expose:
- `conditional_sample(x_obs, obs_mask, n)` — for MC.
- `conditional_characteristic_function(s, x_obs, obs_mask)` — for PCPG.

`GMM` (`gmm.py`) is the reference implementation of that interface: a diagonal-covariance mixture trained by gradient descent on log-likelihood (`fit`), parameterised with `log_vars` to keep variances positive. The conditional CF reweights mixture components by the posterior over k given x_obs, then evaluates each component's Gaussian CF on the missing coordinates only.

Missingness convention: a partially observed input is a 1-D tensor with `NaN` in missing positions; `obs_mask = ~torch.isnan(x_partial)` is the standard idiom used across estimators.

## Dependencies

Not pinned anywhere. The code requires: `torch`, `numpy`, `scikit-learn`, `matplotlib`, `polyagamma` (for PG sampling in `pcpg.py`), and — only for `circuit.py` — `cirkit`.
